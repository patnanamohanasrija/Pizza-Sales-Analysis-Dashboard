# Pizza Sales Analysis Dashboard (Power BI)
This project showcases an end-to-end sales analysis solution built in Power BI using a pizza sales dataset. The dashboard is designed to transform raw transactional data into meaningful business insights through interactive visualizations and dynamic KPIs. It enables users to analyze revenue trends, customer ordering behavior, and product performance across different categories, sizes, and time periods. By highlighting best and worst-selling items, identifying peak sales periods, and uncovering patterns in customer preferences, this dashboard supports data-driven decision-making and provides a clear understanding of overall business performance.

The dashboard provides a comprehensive view of pizza sales data, enabling stakeholders to monitor key performance indicators (KPIs), identify best and worst-performing products, and understand sales trends across time, categories, and sizes.

 # Key Features
Dynamic KPIs
Total Revenue
Average Order Value
Total Pizzas Sold
Total Orders
Average Pizzas per Order
Sales Trend Analysis
Daily order distribution (identify busiest days)
Monthly order trends (seasonality insights)
Category & Size Insights
Sales contribution by pizza category (Classic, Veggie, Chicken, etc.)
Sales distribution by pizza size (Small, Medium, Large)
Best & Worst Sellers
Top 5 pizzas by revenue, quantity, and orders
Bottom 5 pizzas by revenue, quantity, and orders
Interactive Filters
Date range slicer
Pizza category selection

# 🔍 Key Insights
Weekends (especially Friday & Saturday evenings) generate the highest orders
Peak sales months are January and July, indicating seasonal demand patterns
Classic category contributes the highest share of revenue and orders
Large size pizzas dominate overall sales
Certain products consistently underperform, highlighting opportunities for menu optimization
